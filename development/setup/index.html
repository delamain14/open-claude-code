    Development Environment Setup¶

    System Requirements¶

    Required Software¶

    Tool Minimum Version Recommended Version
    Bun 1.0.0 Latest
    Node.js 18.0.0 20.0.0+
    TypeScript 5.0.0 Latest
    Git 2.0.0 Latest
    Python 3.8 3.11+

    Optional Tools¶

    • Visual Studio Code - Recommended code editor
    • Docker - For containerized deployment
    • PostgreSQL - For database support if needed

    Quick Start¶

    1. Clone and Install¶

    git clone https://github.com/your-username/open-claude-code.git
cd open-claude-code
bun install

    2. Configure Environment¶

    cp .env.example .env
# Edit .env, add your API keys

    3. Start Development Server¶

    bun run dev

    VS Code Settings¶

    Create .vscode/settings.json:

    {
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "[typescript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[typescriptreact]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}

    # Run all tests

# Run specific test

# Run tests in watch mode
bun run test --watch

# Generate coverage report
bun run test --coverage

    Workflow¶

    Creating New Feature¶

    1. Create branch 

      git checkout -b feature/your-feature-name

    2. Develop code 

      bun run dev

    3. Run tests 

      bun run test

    4. Code checks 

      bun run lint
bun run format
bun run typecheck

    5. Commit code 

      git add .
git commit -m "feat: add your feature"
git push origin feature/your-feature-name

    6. Create Pull Request

    7. Create PR on GitHub
    8. Wait for CI/CD checks
    9. Wait for code review

    Bug Fixing¶

    1. Create branch 

      git checkout -b fix/bug-name

    2. Locate issue 

      # Enable debug mode
DEBUG=* bun run dev

    3. Write test

    4. Write test to reproduce bug

    5. Then fix code

    6. Verify fix 

      bun run test
bun run lint

    7. Submit PR

    Debug Tips¶

    Enable Verbose Logging¶

    DEBUG=* bun run dev

    }
  ]
}

    FAQ¶

    Q: How to clean build cache?¶

    A: 

    bun cache clean
rm -rf .bun dist build node_modules
bun install

    Q: TypeScript error but code runs?¶

    A: 

    # Update TypeScript
bun update typescript

# Regenerate type definitions
bun run typecheck

    Q: How to handle dependency conflicts?¶

    A: 

    # Update all dependencies
bun update

# Or specific dependency
bun update @types/node

    Q: Hot reload not working?¶

    A: 

    # Restart development server
# Press Ctrl+C to stop, then
bun run dev
